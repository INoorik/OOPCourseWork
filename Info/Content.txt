1. `Bus` - класс, который описывает транспортное средство в автобусном парке, и содержит информацию о марке, модели, 
годе выпуска, номере и других характеристиках автобуса.

2. `Route` - класс, который описывает маршрут, по которому курсирует автобус, и содержит информацию о начальной и 
конечной остановках, длине маршрута, времени отправления и прибытия и других характеристиках маршрута.

3. `Driver` - класс, который описывает водителя, работающего в автобусном парке, и содержит информацию о его имени, 
фамилии, дате рождения, стаже работы и других характеристиках водителя.

4. `Expense` - класс, который описывает расходы, связанные с эксплуатацией автобусов, и содержит информацию о типе 
расхода (например, ремонт, топливо, страхование), дате, сумме и других характеристиках расхода.

5. `Income` - класс, который описывает доходы, связанные с эксплуатацией автобусов, и содержит информацию о типе 
дохода (например, оплата за проезд, реклама), дате, сумме и других характеристиках дохода.

6. `Report` - класс, который содержит методы для выдачи отчетов по запросам, и может включать функции для генерации 
отчетов о доходах и расходах, статистических данных о количестве перевезенных пассажиров и пробеге автобусов.

7. `BusParkContext` - класс контекста базы данных, который используется для работы с данными автобусного парка. Он 
может содержать свойства, которые представляют таблицы базы данных для хранения информации о транспортных средствах, 
маршрутах, водителях, доходах и расходах.

8. `UnitOfWork` -класс, который позволяет объединить несколько операций с базой данных в одну транзакцию. Он может 
содержать методы для сохранения изменений в базе данных, а также для отката (отмены) изменений в случае возникновения ошибок.

9. `BusService` - класс, который содержит методы для работы с данными автобусного парка, такие как создание, чтение, 
обновление и удаление записей в таблицах базы данных. Он может использовать объекты контекста базы данных и UnitOfWork 
для взаимодействия с базой данных.

10. `BusController` - класс, который представляет контроллер в приложении, и обрабатывает запросы от пользователей. 
Он может вызывать методы класса `BusService` для выполнения операций с данными автобусного парка, и возвращать результаты 
выполнения операций в виде представлений для отображения пользователю.

Все эти классы могут использоваться вместе для создания приложения для учета транспортных средств в автобусном парке, 
маршрутов, водителей, учета доходов и расходов, и выдачи отчетов по запросам.
